Windsurf

Codeium 提供のフロー指向エージェント開発ツール。Flow を作って自律的にタスクを進めます。

スクリーンショット

このリポジトリでは実際のスクリーンショット画像を含めていません。

インストール

  1. 公式ドキュメント / 配布ページを確認。
  2. エディタ拡張や Web ベースで提供されている場合は、それに従ってセットアップ。
  3. 必要であれば Codeium の API キーやアカウント情報を設定します。

アカウント作成・ログイン

  1. Codeium のアカウントを作成（メールまたは OAuth）。
  2. Windsurf を有効化し、必要な権限を与える。

基本的な使い方

  1. "New Flow" を作成し、ステップを追加して要件を分割します。
  2. 各ステップに自然言語で目標を記述し、実行ボタンでエージェントを走らせます。
  3. 実行ログを確認し、必要ならステップを調整して再実行します。

注意点

Flow の設計が鍵です。小さく確実に動くステップを作ると安定します。

詳細インストール手順（Windows / Web）

  1. 公式ドキュメントに従い、Windsurf が提供される形式（ブラウザ/拡張/CLI）を確認します。
  2. ブラウザ版: アカウントでログインしてダッシュボードからアクセスします。
  3. 拡張や CLI がある場合は配布ページの手順でインストールします。 
    # 例: プロジェクト作業ディレクトリを開く
cd path\to\project
# エディタで開く
code .
  4. API キーやモデル接続情報を設定（環境変数または UI のシークレット設定）。

実践例 — Flow の作り方

  1. トップレベルの要件を記述（例: "Create a CRUD API for notes with tests"）。
  2. Flow のステップを分割する（例: 初期化、実装、テスト、デプロイ）。
  3. 各ステップに期待する成果物（ファイルパス、テスト要件）を具体的に記述します。
  4. Flow 実行後はローカルでビルド・テストして結果を確認します。

推奨コマンド（ローカル検証用）

# 依存関係のインストール
npm install
# テスト実行
npm test
# ローカルサーバ起動
npm run dev

ベストプラクティスとトラブルシュート

  • Flow は小さく分ける — 一度に多くの作業をさせない。
  • 外部 API やシークレットは事前に設定しておく。
  • Agent の生成物に問題があれば、ログをもとにローカルで再現してデバッグする。
  • 変更履歴を残し、Flow の各実行で差分を追えるようにする。

参考: Codeium の公式ドキュメントを参照してください。
