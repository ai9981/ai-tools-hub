Kiro / Sweep

GitHub の Issue を自動で読み取り、修正 PR を作成することで、ジュニア開発者のワークフローを代行します。

セットアップ

  1. GitHub App またはトークンを発行し、リポジトリへの書き込み権限を付与します。
  2. Webhook を設定して Issue を受け取るようにします。

ワークフロー例

# issue を受けたら自動で調査 → 修正案を PR として作成
# kiro run --issue 123 --repo owner/repo

# 必要に応じて "needs human review" ラベルを付けて手動承認にする

安全運用のポイント

  • 重大な変更は自動マージしない。必ず人のレビューを挟む。
  • ラベルや承認ルールを細かく設定して誤検知や過剰改変を防ぐ。
