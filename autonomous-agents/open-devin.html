OpenDevin / Manus

Devin に対するオープンソース対抗実装。自ホスティングやカスタムポリシー適用が可能です。

セットアップ（Self-hosting）

  1. リポジトリをクローンします（例: git clone https://github.com/open-devin/manus.git）。
  2. 依存関係をインストールし、環境変数を設定します（MANUS_API_KEY など）。
  3. ローカルで動作検証を行い、CI と連携する設定を追加します。

基本的な運用例

# サンプル: ローカルでテストタスクを実行
# manus run --task "Fix lint errors in src/" --repo ./

# テスト環境で動作を確認した後、クラウドや社内サーバーで常時稼働させる

コミュニティと拡張

OSS のため、必要に応じてプラグインやポリシーを追加して安全性と開発スタイルに最適化できる点が強みです。

Settings & SDK（例）

# 環境変数
export MANUS_API_KEY="your_key"

# 実行例（疑似）
# manus run --task "Create tests for utils" --repo ./
