Devin (Cognition)

商用の自律型ソフトウェアエンジニア。要件定義から設計、実装、テスト、デプロイまで自律的に遂行します。

導入と準備

  1. Cognition の提供するポータルでアカウントを作成し、API/組織キーを取得します。
  2. プロジェクトとリポジトリへのアクセス許可（Read/Write）を付与します。
  3. 初期のタスクテンプレート（policy / constraints）を用意します（バジェット・レビュー頻度など）。

基本的な使い方（例）

# Web UI で "Create project: Implement feature X" を入力して計画を生成
# CLI（疑似）
# devin run --task "Implement feature X" --repo https://github.com/your/repo.git --branch devin/feature-x

# デフォルトでプラン → PR 作成 → CI 実行 → マージ申請 の流れを実行します

安全性・ガバナンス

  • 自律性が高いため、レビュー・承認フロー（人間のオーバーサイト）を必ず組み込む。
  • アクセス権限を最小化し、重要な変更（リリース・DBマイグレーション）は手動承認にする。

ユースケースとベストプラクティス

  • 定型的なバグ修正、テスト追加、依存関係の更新などを自動化すると効果が大きい。
  • 安全性の高いサンドボックス環境でまずは少数のレポジトリで試験運用することを推奨。

Settings & Integration（設定例）

# Environment variable
export DEVIN_API_KEY="your_api_key_here"

# Example: run via CLI (pseudo)
# devin run --task "Add unit tests for parseCsv" --repo https://github.com/your/repo.git
