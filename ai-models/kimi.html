Kimi (Moonshot AI)

数百万トークンを処理できる大規模コンテキスト対応モデル。大規模プロジェクトの一括解析に向いています。

導入と接続の注意

非常に大きなコンテキストを扱うため、API の料金・レイテンシ・メモリを考慮して利用する。

利用例

  1. 大規模リポジトリの一括コード検索やリファクタリング提案。
  2. 設計書や仕様書の総合解析、影響範囲の洗い出し。

実装上のヒント

  • 必要に応じて長文分割と要約を組み合わせて入力する。
  • メモリ・コストの可視化と制限を事前に実装する（バッチ処理等）。

Sample Prompts（実例）

  1. リポジトリ全体解析 (日本語): 
    リポジトリを解析し、非推奨 API 'legacyDb.connect' を参照しているモジュールをリストアップしてください。置換案と影響を受けるファイル一覧も示してください。
    Repository analysis (English): 
    Analyze the repository and list modules that reference the deprecated API 'legacyDb.connect'. Provide suggested replacements and affected file list.
  2. 影響範囲の洗い出し (日本語): 
    この仕様書をもとに、変更が必要なすべてのファイルを特定し、作業時間を時間数で見積もってください。
    Impact analysis (English): 
    Given this spec, identify all files that need changes and estimate effort in hours.
  3. 大規模要約 (日本語): 
    添付の 10,000 行の設計ドキュメントを 1 ページのエグゼクティブサマリにまとめ、リスクと対応案をハイライトしてください。
    Large-scale summarization (English): 
    Summarize the attached 10,000-line design document into a 1-page executive summary highlighting risks and action items.

Cost & Performance（目安）

項目目安備考
コスト高大量コンテキスト処理のためコストは高め
レイテンシ高非常に大きな入力は処理時間がかかる
得意分野大規模解析・要約数千〜数百万トークン規模の解析に向く

Settings & SDK（よくある設定）

Kimi は提供元の SDK/エンドポイントに従う（以下は一般的な例）

# 環境変数 (Linux / macOS)
export KIMI_API_KEY="your_api_key_here"
export KIMI_ENDPOINT="https://api.moonshot.ai/v1"

# 例: curl で呼ぶ
curl -X POST "$KIMI_ENDPOINT/generate" -H "Authorization: Bearer $KIMI_API_KEY" -d '{"model":"kimi-large","input":"Summarize the repo"}'
