GPT-4o / o1

OpenAI の汎用モデル群。o1 系は特に「考えてから出力する」戦略で複雑な問題に強い。

導入と接続

  1. OpenAI の API キーを取得し、環境変数に設定します（export OPENAI_API_KEY=xxx）。
  2. 公式 SDK（openai）をインストールしてクライアントを初期化します。

基本的な使い方（Python 例）

from openai import OpenAI
client = OpenAI(api_key=os.environ['OPENAI_API_KEY'])
res = client.responses.create(model='gpt-4o', input='Implement a binary search in Python')
print(res.output_text)

長文・推論時の注意

  • プロンプト設計とコンテキスト管理が重要。必要に応じてシステムプロンプトを分割する。
  • コスト管理のため、max tokens とストリーミングの利用を検討する。

トラブルシューティング

  • レート制限や接続エラーは指数バックオフでリトライすること。
  • API の挙動が変わった場合は OpenAI のリリースノートを確認。

Sample Prompts（実例）

  1. 高度なアルゴリズム実装 (日本語): 
    Python で並列マージソートを実装し、ユニットテストを追加し、時間・空間計算量を説明してください。
    Advanced algorithm implementation (English): 
    Implement a concurrent merge sort in Python with unit tests and explain its time/space complexity.
  2. 設計文書の要約 (日本語): 
    添付の設計書を要約し、上位5つのリスクと緩和案を挙げてください。
    Design doc summarization (English): 
    Summarize the attached design doc and list the top 5 risks with mitigation suggestions.
  3. デバッグ支援 (日本語): 
    この Node.js 関数のバグを見つけ、修正案と再現するテストケースを示してください。
    Debugging assistance (English): 
    Find the bug in this Node.js function and suggest a fix. Include test cases that reproduce the bug.

Cost & Performance（目安）

項目目安備考
コスト高高性能だがトークン単価は高め
レイテンシ中複雑な推論でやや時間がかかる
得意分野複雑な推論・創造的生成設計や難解アルゴリズム向け

Settings & SDK（よくある設定）

例: 環境変数と簡単なクライアントコード

# 環境変数 (Linux / macOS)
export OPENAI_API_KEY="your_api_key_here"

# 環境変数 (Windows PowerShell)
setx OPENAI_API_KEY "your_api_key_here"

# Python (official openai-like pseudo)
from openai import OpenAI
client = OpenAI(api_key=os.environ.get('OPENAI_API_KEY'))
res = client.responses.create(model='gpt-4o', input='Implement a binary search in Python')

# Node.js (pseudo)
const OpenAI = require('openai');
const client = new OpenAI({ apiKey: process.env.OPENAI_API_KEY });
const res = await client.responses.create({ model: 'gpt-4o', input: 'Implement a binary search in Python' });
